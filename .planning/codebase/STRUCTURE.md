## Directory Purposes

**`.planning/codebase/`:**
- Purpose: GSD codebase analysis documents (generated by `gsd-map-codebase` agent)
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md, STACK.md, INTEGRATIONS.md
- Key files: `ARCHITECTURE.md` (system design), `STRUCTURE.md` (this file), others as analyzed
- Committed: Yes (analysis artifacts for downstream GSD commands)

**`build/`:**
- Purpose: Final PDF assembly and publication
- Contains: Python script to render HTML cover + merge comic panels + source paper
- Key files: `build/build-pdf.py` (main entry point); no deps except pypdf, Chrome, system curl
- Committed: Yes
- Generated artifacts: None (output goes to `docs/construct-validity.pdf`)

**`docs/`:**
- Purpose: Published web assets; served by GitHub Pages
- Contains: Static HTML site, rendered comic PNG images, final PDF, QR code
- Key files: `docs/index.html` (hand-authored), `docs/panels/page-*.png` (generated by fal.ai)
- Committed: Yes (site + images; PDF regenerated locally before push)
- Generated artifacts: `docs/panels/*.png`, `docs/construct-validity.pdf`, `docs/qr.png`

**`documentation/`:**
- Purpose: Reference assets and archived versions
- Contains: Old panel versions (e.g., Grok-generated backup), A/B comparison HTML
- Key files: `documentation/explainer-src/manga/construct-validity/` (source source of truth)
- Committed: Yes (part of project history)
- Generated artifacts: `documentation/explainer-src/manga/construct-validity/index.html` (auto-generated from prompts)

**`documentation/explainer-src/manga/construct-validity/`:**
- Purpose: Source authoring for the explainer; single source of truth
- Contains: Character definitions, narrative scripts, image prompts, metadata
- Key files:
  - `manifest.md` — Page table (order, narrative purpose, visual metaphor, status)
  - `character-sheet.md` — Dana & Theo definitions (baked into every page prompt)
  - `page-01.md` through `page-06.md` — Fully self-contained image prompts
  - `notes.md` — Research & ideation (not auto-generated)
  - `README.md` — Project overview
- Committed: Yes (authoring source)
- Generated artifacts: `index.html` (built by `build_prompt_site.py` from prompts)

**`grok/`:**
- Purpose: Panel generation pipeline; text-to-image automation
- Contains: Python script to read prompts, call fal.ai API, save PNG results
- Key files: `generate-panels.py` (main), `requirements.txt` (dependencies), `README.md` (usage)

**`spikes/`:**

## Naming Conventions

**Files:**
- **Page prompts:** `page-0N.md` (zero-padded, e.g., `page-01.md`, `page-02.md`)
- **Rendered panels:** `page-N.png` (e.g., `page-1.png`, `page-2.png`) — without zero-padding
- **Python scripts:** `{verb}-{noun}.py` (e.g., `generate-panels.py`, `build-pdf.py`)
- **Markdown docs:** `{NOUN}.md` or `{adjective}-{noun}.md` (e.g., `README.md`, `character-sheet.md`, `construct-validity-in-vibe-coding.md`)

**Directories:**
- **Functional:** `{noun}/` (e.g., `grok/`, `spikes/`, `build/`)
- **Asset:** `{plural-noun}/` (e.g., `panels/`, `docs/`)
- **Source:** `explainer-src/` (raw content, not generated)

**URLs/Slugs:**
- Topic slug: `construct-validity` (dashes, lowercase; matches directory name)
- Model IDs: fal.ai endpoint format (e.g., `openai/gpt-image-2`, `fal-ai/flux-pro/v1.1`)

## Where to Add New Code

**New Pipeline Tool (e.g., a video generator):**
- Directory: Create `video/` at root level (parallel to `grok/`, `build/`)
- Entry point: `video/generate-video.py` (follow same CLI pattern)
- Docs: `video/README.md` (setup + usage)
- Source: Read from `documentation/explainer-src/manga/construct-validity/page-0N.md` prompts

**New Manga Project (e.g., a second explainer on different paper):**
- Directory: `documentation/explainer-src/manga/{new-slug}/`
- Required files:
  - `manifest.md` — Project metadata (audience, depth, style preset)
  - `character-sheet.md` — Cast definitions
  - `page-01.md` through `page-0N.md` — Image prompts (one per page)
  - `notes.md` — Research & ideation
  - `README.md` — Project overview
- Auto-detection: `generate-panels.py` will discover it under `documentation/explainer-src/manga/` and ask user to pick with `--slug` if multiple projects exist

**New Discussion Resource (e.g., teaching worksheets):**
- Directory: `spikes/` (exploratory resources)
- Naming: `{topic}-{angle}.md` (e.g., `construct-validity-in-vibe-coding.md`)
- No code; standalone markdown

**Local Development Config:**
- File: `.env.local` (create if missing; ignored by `.gitignore`)
- Contents: `FAL_KEY="sk-..."` (from https://fal.ai/dashboard/keys)
- Committed: No (secrets)

**Secrets/Environment:**
- Never commit: `.env`, `.env.local`, `*.key`, `credentials.*`
- Always use: Env var or `.env.local` (loaded by `grok/generate-panels.py` at line 100)
- Aliases: FAL_KEY, FAL_API_KEY, FAL_AI_KEY (checked in order; line 57)
